# Contributing to OOP Tutor

Thank you for your interest in contributing to OOP Tutor! This document provides guidelines for contributing to the project.

## How to Contribute

### Reporting Bugs
- Check if the bug has already been reported in Issues
- Use the bug report template
- Include steps to reproduce the issue
- Specify iOS version and device model
- Attach screenshots if applicable

### Suggesting Enhancements
- Check if the enhancement has already been suggested
- Explain the problem your enhancement would solve
- Provide examples of how it would work
- Consider if it fits the project's scope

### Adding New Lessons
If you'd like to add a new OOP lesson:

1. **Content Requirements:**
   - Clear, concise explanation (200-400 words)
   - Real-world analogies
   - Working code example (preferably Python)
   - 4-6 key points
   - At least 2 quiz questions

2. **Quiz Questions:**
   - Multiple choice with 4 options
   - One correct answer
   - Detailed explanation
   - Test understanding, not memorization

3. **Code Example:**
   - Well-commented
   - Beginner-friendly
   - Demonstrates the concept clearly
   - Follows Python best practices

### Pull Request Process

1. Fork the repository
2. Create a new branch (`git checkout -b feature/your-feature-name`)
3. Make your changes
4. Test thoroughly on iOS device/simulator
5. Commit with clear, descriptive messages
6. Push to your fork
7. Open a Pull Request

### Code Style Guidelines

**Swift Code:**
- Follow Swift API Design Guidelines
- Use SwiftUI best practices
- Keep views small and focused
- Use meaningful variable names
- Add comments for complex logic

**Project Structure:**
- Keep models separate from views
- Use proper file organization
- Don't mix UI and business logic

### Testing Your Changes

Before submitting a PR:
- [ ] Build succeeds without errors
- [ ] App runs on iOS 17+ devices
- [ ] UI looks correct on iPhone and iPad
- [ ] All interactive elements work
- [ ] Progress tracking works correctly
- [ ] Quiz scoring is accurate
- [ ] No crashes or major bugs

### Content Guidelines

**Lessons should:**
- Be educational and accurate
- Use simple, clear language
- Include practical examples
- Be beginner-friendly
- Stay focused on one concept

**Avoid:**
- Overly complex explanations
- Incorrect technical information
- Offensive or inappropriate content
- Plagiarized content

## Code of Conduct

### Our Pledge
We are committed to providing a welcoming and inclusive environment for all contributors.

### Expected Behavior
- Be respectful and professional
- Accept constructive criticism
- Focus on what's best for the project
- Show empathy towards others

### Unacceptable Behavior
- Harassment or discrimination
- Trolling or insulting comments
- Publishing others' private information
- Other unprofessional conduct

## Questions?

If you have questions about contributing:
- Open a Discussion on GitHub
- Check existing Issues and Pull Requests
- Review the README for project information

## Recognition

Contributors will be acknowledged in the project. Significant contributions may be highlighted in release notes.
